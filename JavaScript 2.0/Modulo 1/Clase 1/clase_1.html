Explicación detallada por subtema

2.1 · Desarrollo web y Full Stack

Una aplicación web moderna se divide en dos grandes partes que se comunican entre sí:

Frontend: lo que el usuario ve y con lo que interactúa en el navegador (interfaz, botones, formularios).
Backend: la lógica que corre en un servidor, procesa datos, aplica reglas de negocio y habla con la base de datos.

Full Stack significa que un mismo conjunto de tecnologías puede trabajar en ambas partes. El Stack MEAN es "full stack" porque usa JavaScript tanto en frontend como en backend — no necesitas aprender dos lenguajes distintos.

2.2 · El Stack MEAN

LetraTecnologíaRol en la aplicación
MMongoDBBase de datos NoSQL — documentos tipo JSON, no tablas rígidas
EExpressFramework de backend sobre Node.js — rutas y reglas del servidor
AAngularFramework de frontend — construye la interfaz del usuario
NNode.jsEntorno de ejecución — corre JavaScript fuera del navegador

Node.js es la base sobre la que corre Express; Angular corre en el navegador del usuario; MongoDB es el almacenamiento al que Express se conecta.

2.3 · Node.js — instalación y verificación

Permite ejecutar JavaScript fuera del navegador. Se instala la versión LTS (Long Term Support) desde nodejs.org — más estable que "Current".

# Verificación en terminal
node -v
npm -v

npm (Node Package Manager) viene incluido con Node y gestiona paquetes del ecosistema JS.

2.4 · VS Code

Editor de código estándar de la industria. Extensiones instaladas en esta clase:

Live Server Recarga automática de la página al guardar cambios en HTML/CSS/JS.
Prettier Formatea el código automáticamente (indentación, comillas, espacios).

2.5 · Git

Sistema de control de versiones: guarda un historial de cambios para retroceder, comparar y trabajar en equipo sin sobrescribir el trabajo de otros.

# Configuración inicial, una sola vez por computador
git config --global user.name "Tu Nombre"
git config --global user.email "tu.email@ejemplo.com"

No crea ningún proyecto — solo le dice a Git quién eres, para que cada commit quede firmado.

2.6 · Angular CLI

Genera automáticamente toda la estructura base de un proyecto Angular.

# Instalación global
npm install -g @angular/cli

# Proyecto de prueba
mkdir prueba-cli
cd prueba-cli
ng new mi-app --minimal

El flag --minimal genera una versión reducida, ideal para una primera exploración.

2.7 · MongoDB Atlas

En vez de instalar MongoDB localmente, se usa MongoDB Atlas: MongoDB alojado en la nube, con capa gratuita.

Pasos cubiertos: crear cuenta gratuita → crear un clúster → obtener la cadena de conexión (connection string) que Express usará más adelante.

Dato sensible La cadena de conexión contiene usuario/contraseña — no se sube a repositorios públicos.

04

¿Para qué sirve cada concepto?

  • Node.js → correr JavaScript como backend, sin otro lenguaje.
  • VS Code + extensiones → acelera el flujo de trabajo diario.
  • Git → indispensable para trabajar en equipo sin perder trabajo ajeno.
  • Angular CLI → evita configurar manualmente un proyecto desde cero.
  • MongoDB Atlas → base de datos real, accesible desde cualquier lugar, sin mantenimiento propio.

Ejemplos abstractos

git config --global user.name "[tu_nombre]"
git config --global user.email "[tu_correo]"

git config --global user.name/user.email son fijos. [tu_nombre] y [tu_correo] los defines tú.

mkdir [nombre_carpeta]
cd [nombre_carpeta]
ng new [nombre_app] --minimal

mkdir, cd, ng new, --minimal son fijos. Los nombres entre corchetes son tuyos y se repiten según el proyecto.

node -v
npm -v

Estos comandos son totalmente fijos — sin ningún valor reemplazable.

06

Excepciones y casos especiales

Node LTS vs CurrentInstalar "Current" en vez de "LTS" puede traer cambios recientes que rompan compatibilidad con paquetes usados en clase.
npm install -gSi olvidas el flag -g, Angular CLI solo queda instalado en la carpeta actual y ng no funcionará en otro lugar.
Git config sin --globalSin --global, la configuración solo aplica al proyecto actual — "en otro proyecto no aparece mi nombre".
MongoDB Atlas free tierLímites de almacenamiento y rendimiento — suficiente para aprender, no para producción real.
Connection string expuestaCompartirla en un repositorio público es un riesgo de seguridad real.

Metáforas y ejemplos visuales

08

Errores típicos de un junior

  • Instalar Node.js y olvidar reiniciar la terminal antes de verificar con node -v.
  • Instalar Angular CLI sin -g y no entender por qué ng "no existe" en otra carpeta.
  • Olvidar configurar Git antes del primer commit en clases futuras.
  • Perder la cadena de conexión de MongoDB Atlas y tener que regenerarla.
  • Confundir la carpeta de prueba con la carpeta del proyecto real, mezclando archivos.
