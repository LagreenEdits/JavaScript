Explicación detallada por subtema

2.1 · ¿Qué es HTML y qué es CSS?

HTML (HyperText Markup Language) no es un lenguaje de programación: es un lenguaje de marcado. Describe la estructura y el significado del contenido — "esto es un título", "esto es una lista" — sin decidir colores ni tamaños.

CSS (Cascading Style Sheets) decide cómo se ve ese contenido: colores, tipografía, espaciado, posición. "Cascading" (en cascada) es clave: los estilos se combinan según reglas de prioridad (ver especificidad, 2.15).

Por qué se separan: mezclar estructura y presentación obliga a tocar el HTML por cada cambio visual. Separarlos permite cambiar el "traje" (CSS) sin tocar el "esqueleto" (HTML) — base de la accesibilidad y el SEO.

2.2 · Anatomía de un tag HTML

<a href="https://escuelanew.com">Visita mi Web</a>

Elemento: la etiqueta completa, apertura + contenido + cierre.
Etiqueta de apertura: <a href="..."> · Nombre del atributo: href · Valor del atributo: "https://escuelanew.com"
Contenido: Visita mi Web · Etiqueta de cierre: </a>

Con contenido (necesitan apertura y cierre): <p>Texto</p>. Auto-cerradas (sin cierre): <img />, <br />, <hr />.

2.3 · Estructura base de un documento HTML

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Mi primera página</title>
  </head>
  <body>
    <h1>Hola mundo</h1>
  </body>
</html>

<!DOCTYPE html> le dice al navegador "interpreta esto como HTML5"; sin ella puede activarse "modo quirks" y romper el CSS. <head> guarda metadatos (no se muestra); <body> contiene todo el contenido visible.

2.4 · Meta etiquetas esenciales

<meta charset="UTF-8" /> define la codificación de caracteres (sin ella, tildes y "ñ" pueden verse rotas). <meta name="viewport" ...> es indispensable para que la página se vea bien en móviles. <title> aparece en la pestaña del navegador y en resultados de buscadores.

2.5 · Etiquetas de texto

EtiquetaUsoNota
h1–h6Encabezados, mayor a menor jerarquíaIdealmente un solo h1 por página
pPárrafoBloque, ocupa toda la línea
spanContenedor en línea sin significado semánticoPara estilizar parte de un texto
strongImportancia fuerteSemántico + negrita por defecto
emÉnfasisSemántico + cursiva por defecto
Diferencia clavestrong/em tienen significado semántico (los lectores de pantalla los acentúan); b/i solo cambian la apariencia, sin significado.

2.6 · Etiquetas semánticas de estructura

<header>Cabecera del sitio o de una sección</header>
<nav>Menú de navegación</nav>
<main>Contenido principal (uno solo por página)</main>
<section>Agrupa contenido relacionado</section>
<article>Contenido autónomo y reutilizable</article>
<aside>Contenido secundario (ej. sidebar)</aside>
<footer>Pie de página o de sección</footer>

Antes de HTML5 todo era div: un div no dice nada sobre el contenido. Las etiquetas semánticas resuelven esto — el nombre de la etiqueta ya comunica su función (accesibilidad, SEO).

2.7 · Listas

<ul>
  <li>Elemento sin orden</li>
</ul>

<ol>
  <li>Primer paso</li>
</ol>

ul (unordered): viñetas, el orden no importa. ol (ordered): numerada, el orden sí importa. li siempre dentro de ul u ol.

2.8 · Enlaces

<a href="https://ejemplo.com" target="_blank" rel="noopener">Ir al sitio</a>
<a href="#seccion2">Ir a la sección 2</a>
<a href="mailto:correo@ejemplo.com">Escríbenos</a>

href puede ser una URL externa, un ancla (#id) o un protocolo especial (mailto:, tel:). target="_blank" abre en nueva pestaña; se recomienda acompañarlo de rel="noopener" por seguridad.

2.9 · Imágenes

<img src="logo.png" alt="Logo de la empresa" width="200" height="100" />

src: ruta del archivo. alt: texto alternativo, obligatorio por accesibilidad y porque se muestra si la imagen no carga. Es auto-cerrada, no lleva </img>.

2.10 · Formularios básicos

<form action="/enviar" method="POST">
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre" placeholder="Escribe tu nombre" />

  <label for="correo">Correo:</label>
  <input type="email" id="correo" name="correo" required />

  <input type="checkbox" id="acepto" name="acepto" />
  <label for="acepto">Acepto términos</label>

  <button type="submit">Enviar</button>
</form>

Tipos de input comunes: text, email, password, number, checkbox, radio, date, file, submit.

ClaveEl valor de label for="..." debe coincidir con el id del input correspondiente — así, al hacer clic en el texto, el foco va al input.

2.11 · Atributos globales

id: identificador único en todo el documento. class: puede repetirse, y un elemento puede tener varias separadas por espacio: class="tarjeta destacada". style: CSS directo en el elemento, máxima prioridad en la cascada. title: tooltip al pasar el mouse.

2.12 · Formas de vincular CSS

/* 1. Inline */
<p style="color: blue;">Texto azul</p>

/* 2. Interno, dentro de <head> */
<style>
  p { color: blue; }
</style>

/* 3. Externo — la mejor práctica */
<link rel="stylesheet" href="styles.css" />

Por qué externo: el navegador puede cachearlo (se reutiliza en varias páginas), el código queda organizado, y varios HTML pueden compartir el mismo diseño.

2.13 · Sintaxis básica de CSS

selector {
  propiedad: valor; /* declaración */
}

Selector + bloque de declaraciones = regla CSS. Ej: p { color: navy; font-size: 16px; }.

2.14 · Selectores CSS

/* Por etiqueta: TODOS los <p> */
p { color: black; }

/* Por clase: class="destacado" */
.destacado { color: orange; }

/* Por id: único */
#encabezado { background-color: gray; }

/* Universal: TODOS los elementos */
* { margin: 0; padding: 0; }

/* Descendiente: <li> dentro de <ul> */
ul li { list-style: none; }

2.15 · Especificidad CSS

Cuando dos reglas se contradicen, gana la de mayor especificidad (no la escrita después, salvo empate).

p { color: black; }            // especificidad baja
.destacado { color: orange; }  // gana sobre "p"
#titulo { color: red; }        // gana sobre ".destacado"

Orden de mayor a menor peso: style inline → #id → .clase → etiqueta → * universal. !important fuerza a ganar sin importar especificidad, pero se considera mala práctica.

2.16 · Herencia de estilos

Algunas propiedades heredan automáticamente a los hijos (color, font-family, font-size, text-align). Otras no heredan (margin, padding, border, width, height) porque no tendría sentido que un hijo copie el tamaño de caja de su padre.

2.17 · Propiedades de texto

p {
  color: #333333;
  font-size: 18px;
  font-family: "Segoe UI", Arial, sans-serif;
  font-weight: bold;
  text-align: center;
}

font-family es una lista de preferencia: si el navegador no tiene la primera fuente, prueba la siguiente; el genérico final (sans-serif) es el respaldo garantizado.

2.18 · Modelo de caja (Box Model)

┌─────────────────────────────┐ │ margin │ ← espacio FUERA del elemento │ ┌───────────────────────┐ │ │ │ border │ │ ← borde del elemento │ │ ┌─────────────────┐ │ │ │ │ │ padding │ │ │ ← espacio interno │ │ │ ┌───────────┐ │ │ │ │ │ │ │ content │ │ │ │ ← contenido real │ │ │ └───────────┘ │ │ │ │ │ └─────────────────┘ │ │ │ └───────────────────────┘ │ └─────────────────────────────┘
.caja {
  width: 200px;
  height: 100px;
  padding: 20px;
  border: 2px solid black;
  margin: 10px;
}

2.19 · box-sizing: content-box vs border-box

.caja-default {
  box-sizing: content-box; /* valor por defecto */
  width: 200px;
  padding: 20px;
  border: 5px solid black;
  /* ancho REAL = 200 + 20+20 (padding) + 5+5 (border) = 250px */
}

.caja-border-box {
  box-sizing: border-box;
  width: 200px;
  padding: 20px;
  border: 5px solid black;
  /* ancho REAL = 200px, se descuenta del content */
}

📌 Práctica común en la industria: * { box-sizing: border-box; } para que los cálculos de tamaño sean predecibles.

2.20 · Unidades de medida

UnidadTipoSe calcula respecto a...
pxAbsolutaPíxeles fijos, no cambia
%RelativaEl tamaño del elemento padre
emRelativaEl font-size del padre
remRelativaEl font-size de la raíz (html)
html { font-size: 16px; }
.padre { font-size: 20px; }
.hijo-em { font-size: 1.5em; }   /* 1.5 × 20px = 30px */
.hijo-rem { font-size: 1.5rem; } /* 1.5 × 16px = 24px, sin importar el padre */

2.21 · Colores

.a { color: red; }                    /* nombre predefinido */
.b { color: #ff0000; }                /* hexadecimal */
.c { color: rgb(255, 0, 0); }         /* rgb: 0-255 por canal */
.d { color: rgba(255, 0, 0, 0.5); }   /* rgba: + alpha 0 a 1 */
red #ff0000 rgb(255,0,0) rgba(255,0,0,0.5)

2.22 · Buenas prácticas y organización

Separar HTML de CSS en archivos externos · usar etiquetas semánticas en vez de div genéricos · nombrar clases por propósito, no por apariencia (.card-destacada mejor que .texto-rojo) · indentar consistentemente · comentar bloques complejos.

04

¿Para qué sirve cada concepto?

  • DOCTYPE + estructura base → garantiza que el navegador renderice de forma consistente y moderna, evitando modo "quirks".
  • Meta viewport → hace que la web sea usable en celulares, requisito básico de cualquier sitio actual.
  • Etiquetas semánticas → mejoran el SEO y la accesibilidad para lectores de pantalla.
  • class vs id → class reutiliza el mismo estilo en múltiples elementos; id apunta a un elemento único (ej. para JavaScript o un ancla).
  • CSS externo → mantiene un solo archivo de estilos para todo un sitio, facilitando cambios globales.
  • Especificidad → evita el clásico "puse el estilo pero no se aplica".
  • Box model / box-sizing → fundamental para maquetar sin sorpresas de tamaño.
  • Unidades relativas (rem, %) → permiten diseños responsivos sin recalcular todo a mano.
  • Formularios → puerta de entrada para que el usuario envíe datos (login, contacto, registro).

Ejemplos abstractos

<!DOCTYPE html>
<html>
  <head></head>
  <body>
    <h1></h1>
    <p></p>
    <ul><li></li></ul>
    <a href=""></a>
    <img src="" alt="" />
  </body>
</html>

html, head, body, h1...h6, p, ul, li, a, img, form, input, label son nombres de etiquetas fijos, no se inventan. href, src, alt, type, for son nombres de atributos fijos. color, margin, padding, width, height, font-size, box-sizing son propiedades CSS fijas.

<div class="[nombre_clase_1]" id="[id_unico_1]">
  <p class="[nombre_clase_1]">Repite la misma clase, es válido</p>
</div>
.[nombre_clase_1] { color: [valor_color_1]; }
#[id_unico_1] { padding: [valor_medida_1]; }

[nombre_clase_1] se puede repetir en múltiples elementos y el estudiante elige el nombre. [id_unico_1] lo elige el estudiante, pero no se puede repetir: cada id debe ser único en todo el documento. [valor_color_1], [valor_medida_1] son valores libres (ej. red, 20px), deben respetar el formato válido de la propiedad.

Elemento¿Se puede renombrar?¿Se puede repetir?
Nombre de etiqueta (div, p, h1...)✕ No, fijo del lenguaje✓ Sí, todas las que quieras
Nombre de atributo (href, src, class...)✕ No, fijo del lenguaje✓ Sí, en distintos elementos
Valor de class✓ Sí, lo elige el estudiante✓ Sí, en múltiples elementos
Valor de id✓ Sí, lo elige el estudiante✕ No, debe ser único
Nombre de propiedad CSS (color, margin...)✕ No, fijo del lenguaje✓ Sí, en múltiples reglas
Valor de una propiedad CSS (red, 20px...)✓ Sí, dentro de valores válidos✓ Sí, en múltiples reglas
06

Excepciones y casos especiales

id único vs class repetibleSi por error se repite un id, la página sigue "funcionando" visualmente, pero es inválido y puede causar bugs graves en JavaScript (getElementById solo devuelve el primero) y fallar en validadores HTML.
Especificidad no es "quién está más abajo"A igual especificidad (dos clases), gana el escrito después. Si uno tiene mayor especificidad (id vs clase), gana ese sin importar el orden.
Herencia no aplica a todoPropiedades de "caja" (margin, padding, border, width) no heredan; propiedades de texto (color, font-family) sí heredan — confusión frecuente en principiantes.
box-sizing cambia el layoutEl mismo CSS con content-box vs border-box produce cajas de tamaño final distinto; por eso frameworks como Bootstrap o Tailwind fuerzan border-box globalmente.
em vs remAnidar varios elementos con em produce un efecto acumulativo en cascada; rem evita ese efecto al ser siempre relativo a la raíz.
Etiquetas auto-cerradasimg, br, input, meta no tienen contenido ni cierre; intentar cerrarlas con </img> es un error.
label for depende del idSi el for no coincide con ningún id, el label pierde su función de accesibilidad — falla silenciosa, sin error visible.
h1 único por páginaNo está limitado técnicamente en la especificación actual, pero sigue siendo convención recomendada de SEO/accesibilidad; esto puede variar según tu plataforma.
07

Metáforas y ejemplos visuales

HTML vs CSSHTML es el esqueleto y los órganos de un cuerpo (qué es cada parte); CSS es la ropa y el maquillaje (cómo se ve por fuera).
class vs idclass es como el uniforme escolar: muchos lo comparten. id es como el número de cédula: cada persona tiene el suyo, y no se repite.
EspecificidadUn torneo de pesos: style inline es peso pesado, id es peso medio-alto, class es peso medio, etiqueta simple es peso ligero. Gana el de mayor peso, sin importar quién entró primero.
Herencia CSSLos rasgos genéticos de una familia: el color de ojos (color, font-family) se hereda de padres a hijos, pero el tamaño de la casa (margin, width) no se hereda automáticamente.
Box ModelUna caja de regalo dentro de una caja de envío: el regalo es el content, el papel burbuja es el padding, la caja de cartón es el border, y el espacio libre en el camión es el margin.
box-sizing: border-boxDecir "el tamaño de la maleta ya incluye ruedas y asas", en vez de "el tamaño es solo el compartimento interno y las ruedas se suman aparte".
em vs remem es medir tu altura relativa a la persona que tienes al lado (cambia según quién esté cerca); rem es medirte siempre contra una regla fija en la pared, sin importar quién esté al lado.
CSS externo vs inlineCSS externo es un manual de estilo corporativo único para toda la empresa; CSS inline es cada empleado decidiendo su propio uniforme cada mañana: funciona, pero no escala.
08

Errores típicos de un junior

  • Olvidar cerrar etiquetas (<p>Texto sin </p>), rompiendo el anidamiento del resto del documento.
  • Confundir class con id: usar id para algo que se repite (ej. 10 tarjetas con el mismo id), generando HTML inválido.
  • No vincular bien el CSS externo: rutas relativas mal escritas en <link href="...">, o el link fuera de <head>.
  • No entender el box model: sorprenderse de que un elemento con width: 100px ocupe más espacio del esperado por no considerar padding/border.
  • Usar <div> para todo en vez de etiquetas semánticas, perdiendo accesibilidad y SEO.
  • Olvidar el atributo alt en imágenes.
  • Escribir valores CSS sin unidad cuando es obligatoria (margin: 10; en vez de margin: 10px;), lo cual el navegador ignora silenciosamente.
  • Anidar <li> fuera de <ul>/<ol>, o texto suelto directo dentro de <ul>.
  • Abusar de !important en vez de entender por qué la especificidad no funcionaba.
  • No usar <label for="..."> correctamente en formularios, perjudicando la accesibilidad.
