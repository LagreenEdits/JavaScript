Explicación detallada por subtema

2.1 · Qué son las estructuras de control y por qué existen

Sin estructuras de control, un programa se ejecutaría siempre igual, línea por línea, sin poder "decidir" ni "repetir". Las estructuras de control dan dos superpoderes:

Decisión: ejecutar un bloque de código u otro según si una condición es verdadera o falsa (if, switch, ternario).
Repetición: ejecutar un mismo bloque varias veces sin copiarlo y pegarlo manualmente (for, while, do...while, for...of, for...in).

2.2 · Condicional if / else if / else

Uso: tomar decisiones basadas en condiciones booleanas.

const n = 7;
if (n > 0) {
  console.log('positivo');
} else if (n < 0) {
  console.log('negativo');
} else {
  console.log('cero');
}

Cómo funciona internamente: JS evalúa las condiciones de arriba hacia abajo. En cuanto una es "truthy", ejecuta ese bloque y se salta el resto — los demás else if/else ni se evalúan. Si ninguna es verdadera, corre el else (si existe).

Buenas prácticas: evitar condiciones complejas en una línea (extraer a funciones con nombre); preferir === salvo que se requiera coerción; un if sin else es válido y común.

2.3 · Operador ternario

Uso: una forma corta de if/else simple que devuelve un valor, ideal para asignaciones rápidas.

const edad = 20;
const mensaje = (edad >= 18) ? 'Eres mayor de edad' : 'Eres menor de edad';
console.log(mensaje); // "Eres mayor de edad"

Sintaxis: condicion ? valorSiTrue : valorSiFalse. A diferencia de if/else, el ternario es una expresión que produce un valor utilizable.

Cuándo usarlo: ternario → cuando el resultado es un valor a asignar directamente (clase CSS, texto, número). if/else → cuando necesitas ejecutar varias líneas o efectos secundarios.

Ejemplo real de un ejercicio de esta clase, usando el ternario dentro de un bucle:

const container = document.getElementById('container');
for (let i = 1; i <= 10; i++) {
  const type = (i % 2 === 0) ? 'par' : 'impar';
  const div = document.createElement('div');
  div.classList.add(`square-${type === 'par' ? 'red' : 'blue'}`);
  div.textContent = i;
  container.appendChild(div);
}

Aquí hay dos ternarios anidados dentro de un for: uno decide "par"/"impar", el otro decide la clase CSS. Anidar es válido, pero más de un nivel reduce la legibilidad.

2.4 · Operadores lógicos y short-circuit evaluation

const edad = 25;
const tieneCedula = true;
if (edad >= 18 && tieneCedula) {
  console.log('Puede votar');
}

const esFeriado = false;
const esFinDeSemana = true;
if (esFeriado || esFinDeSemana) {
  console.log('No hay clases hoy');
}

const estaLloviendo = false;
if (!estaLloviendo) {
  console.log('Puedes salir sin paraguas');
}

Short-circuit evaluation: JavaScript no evalúa más de lo necesario. Con &&, si el primer operando es falso, ni evalúa el segundo. Con ||, si el primero es verdadero, ni evalúa el segundo.

function log(msg) {
  console.log(msg);
  return true;
}

false && log('esto NO se imprime'); // el && corta antes de log()
true || log('esto tampoco se imprime'); // el || corta antes de log()

Base de patrones comunes como usuario && usuario.nombre (evitar errores si es null) o valor || 'valor por defecto'.

2.5 · switch / case / default / break

Uso: seleccionar entre múltiples caminos discretos basados en un mismo valor.

const dia = 'viernes';
switch (dia) {
  case 'lunes':
    console.log('inicia la semana');
    break;
  case 'viernes':
    console.log('fin de semana cerca');
    break;
  default:
    console.log('día normal');
}

JS compara dia con cada case usando igualdad estricta (===). Al encontrar coincidencia, ejecuta ese bloque y sigue ejecutando los siguientes a menos que haya un break.

Fall-through no intencional Si olvidas el break, la ejecución "cae" al siguiente case sin verificar su condición. 
switch (dia) {
  case 'sabado':
  case 'domingo':
    console.log('es fin de semana'); // se ejecuta para AMBOS casos
    break;
  default:
    console.log('es día de semana');
}

El fall-through también puede ser intencional, como arriba, para compartir un mismo bloque entre varios case.

2.6 · Bucle for clásico

Uso: iteraciones donde conoces (o controlas) el índice/contador.

for (let i = 0; i < 5; i++) {
  console.log(i);
}

Anatomía (tres partes separadas por ;):

1. Inicialización (let i = 0): se ejecuta una sola vez.
2. Condición (i < 5): se evalúa antes de cada iteración.
3. Incremento (i++): se ejecuta después de cada iteración.

Caso de uso: recorrer arrays con índice, construir elementos del DOM con un contador.

2.7 · Bucle while

Uso: repetir mientras una condición sea verdadera; la condición se evalúa antes de cada iteración.

let i = 0;
while (i < 3) {
  console.log(i);
  i++;
}

A diferencia del for, no tiene sintaxis fija de tres partes — ideal cuando no sabes de antemano cuántas veces se repetirá.

PeligroSi la condición nunca se vuelve falsa (ej. olvidas el i++), el bucle se ejecuta para siempre.

2.8 · Bucle do...while

Uso: idéntico a while, pero garantiza que el cuerpo se ejecute al menos una vez.

let i = 0;
do {
  console.log(i);
  i++;
} while (i < 3);

En while, la condición se evalúa antes del primer bloque; en do...while, se evalúa después.

let i = 10;

while (i < 5) {
  console.log('while:', i); // nunca se imprime
  i++;
}

do {
  console.log('do...while:', i); // se imprime UNA vez
  i++;
} while (i < 5);

Caso de uso: menús interactivos o validaciones donde necesitas mostrar/pedir algo al menos una vez.

2.9 · for...of

Uso: iterar directamente sobre los valores de un iterable (arrays, strings, Map, Set, NodeList, etc.).

const frutas = ['manzana', 'pera', 'uva'];
for (const f of frutas) {
  console.log(f);
}

Ventaja: sintaxis limpia cuando no necesitas el índice. También funciona sobre strings:

for (const letra of 'Hola') {
  console.log(letra); // H, o, l, a
}

2.10 · for...in

Uso: iterar sobre las claves (propiedades enumerables) de un objeto.

const obj = { a: 1, b: 2 };
for (const key in obj) {
  console.log(key, obj[key]);
}
Advertenciafor...in itera propiedades enumerables, incluyendo las heredadas. Para arrays no se recomienda — usar for o for...of.

2.11 · break y continue dentro de bucles

break: termina el bucle por completo, de inmediato.
continue: salta solo la iteración actual y continúa con la siguiente.

// break: detener apenas se encuentra el número
for (let i = 0; i < 10; i++) {
  if (i === 5) break;
  console.log(i); // imprime 0, 1, 2, 3, 4 y se detiene
}

// continue: saltar solo los números pares
for (let i = 0; i < 6; i++) {
  if (i % 2 === 0) continue;
  console.log(i); // imprime 1, 3, 5
}

2.12 · Bucles anidados

Uso: un bucle dentro de otro — el interno se ejecuta completo por cada iteración del externo.

for (let fila = 1; fila <= 3; fila++) {
  for (let columna = 1; columna <= 3; columna++) {
    console.log(`Fila ${fila}, Columna ${columna}`);
  }
}
// Se ejecuta 3 x 3 = 9 veces en total

Caso de uso: matrices, tableros, cuadrículas de elementos del DOM, comparar dos listas.

Nota de rendimiento: si el externo corre N veces y el interno M veces, el total es N × M — con listas grandes puede ser lento.

2.13 · Bucles infinitos

Ocurren cuando la condición de parada nunca se vuelve falsa.

// 1. Olvidar el incremento/decremento
let i = 0;
while (i < 5) {
  console.log(i);
  // falta i++  →  bucle infinito, i siempre es 0
}

// 2. Modificar la variable equivocada dentro del bucle
let contador = 0;
while (contador < 5) {
  let otraVariable = contador + 1; // esto NO modifica "contador"
  console.log(otraVariable);
  // contador nunca cambia → bucle infinito
}

Cómo evitarlos: verifica que la variable de la condición se modifique dentro del cuerpo; pregúntate "¿en qué punto esta condición se vuelve falsa?" antes de ejecutar.

2.14 · Buenas prácticas: qué bucle usar según el caso

SituaciónBucle recomendado
Sé exactamente cuántas veces repetirfor
Necesito solo los valores de un array/iterablefor...of
Necesito las claves de un objetofor...in
No sé cuántas veces se repetiráwhile
Necesito ejecutar el bloque al menos una vez sí o sído...while
Recorrer transformando/filtrando/sumandoMétodos funcionales (map, filter, reduce)

Regla general: evita for...in en arrays; prefiere for, for...of o métodos funcionales.

2.15 · Aplicación práctica: condicionales + bucles en el DOM

La combinación más común en desarrollo web real es un bucle que genera elementos, decidiendo con un condicional cómo se ve cada uno:

const container = document.getElementById("container");
for (let i = -100; i <= 100; i++) {
  const square = document.createElement("div");
  if (i % 2 === 0) {
    square.classList.add("square-even");
    square.textContent = `🤝 + ${i}`;
  } else {
    square.textContent = `🫂 + ${i}`;
    square.classList.add("square-odd");
  }
  container.appendChild(square);
}

Combina un for clásico que controla exactamente el rango (-100 a 100), un if/else dentro del bucle que decide clase CSS y texto por iteración, y manipulación del DOM (createElement, classList.add, textContent, appendChild) que se profundiza en la Clase 5.

Compáralo con el ejemplo del ternario (2.3): ambos resuelven el mismo problema (pares/impares con clase CSS distinta), pero uno usa if/else con varias líneas y el otro un ternario en una sola línea.

04

¿Para qué sirve cada concepto?

  • if / else if / else → tomar decisiones simples o encadenadas basadas en condiciones.
  • Operador ternario → asignaciones condicionales cortas y legibles en una línea (clases CSS, textos dinámicos).
  • Operadores lógicos + short-circuit → condiciones compuestas y patrones seguros como valores por defecto o acceso protegido.
  • switch → reemplazar cadenas largas de else if cuando comparas la misma variable contra múltiples valores discretos.
  • for → repetir algo un número exacto de veces, con control total sobre el contador.
  • while / do...while → repetir algo mientras se cumpla una condición desconocida de antemano.
  • for...of → recorrer valores de arrays/iterables de forma limpia.
  • for...in → inspeccionar las propiedades (claves) de un objeto dinámicamente.
  • break / continue → afinar el control dentro de un bucle sin reescribir toda su lógica.
  • Bucles anidados → resolver problemas que requieren combinar dos dimensiones de datos.
  • Condicionales + bucles en el DOM → generar interfaces dinámicas cuyo estilo o contenido cambia según cada elemento.

Ejemplos abstractos

if ([condicion_1]) {
  [instruccion_1];
} else if ([condicion_2]) {
  [instruccion_2];
} else {
  [instruccion_3];
}

if, else if, else son palabras reservadas, fijas. { } son símbolos de cierre, fijos. [condicion_1], [condicion_2] las define el estudiante y se pueden repetir tantos else if como se necesiten.

const [variable_resultado] = ([condicion]) ? [valor_si_true] : [valor_si_false];

Los símbolos ? y : son sintaxis fija del ternario. [variable_resultado], [condicion], [valor_si_true], [valor_si_false] los define el estudiante.

switch ([variable_a_comparar]) {
  case [valor_1]:
    [instruccion_1];
    break;
  case [valor_2]:
    [instruccion_2];
    break;
  default:
    [instruccion_por_defecto];
}

switch, case, default, break son palabras reservadas, fijas; el : tras cada case/default es obligatorio. [variable_a_comparar], [valor_1], [valor_2] los define el estudiante — se pueden agregar tantos case como se necesiten.

for ([inicializacion]; [condicion]; [incremento]) {
  [instruccion_del_bucle];
}

for y los dos ; internos son sintaxis fija, siempre en ese orden. [inicializacion], [condicion], [incremento] los define el estudiante (típicamente let i = 0, i < N, i++, pero el nombre puede cambiar).

while ([condicion]) {
  [instruccion_del_bucle];
}

do {
  [instruccion_del_bucle];
} while ([condicion]);

while, do son palabras reservadas, fijas. El ; final tras while([condicion]) en do...while es obligatorio y fijo — a diferencia del while normal.

for (const [nombre_elemento] of [iterable]) {
  [instruccion_del_bucle];
}

for (const [nombre_clave] in [objeto]) {
  [instruccion_del_bucle];
}

for, of, in, const son palabras reservadas, fijas. [nombre_elemento], [nombre_clave] los elige el estudiante. [iterable], [objeto] son la variable a recorrer, definida previamente.

06

Excepciones y casos especiales

Fall-through intencionalVarios case seguidos sin break entre ellos, compartiendo un mismo bloque de código, es una técnica válida (ej. agrupar "sabado" y "domingo"). El error solo ocurre cuando el fall-through no es intencional.
do...while con condición falsa desde el inicioEl cuerpo se ejecuta igual una vez, aunque la condición ya sea falsa antes de empezar — a diferencia de while, que en ese caso no ejecutaría nunca el cuerpo.
Bucles infinitos silenciososUna condición que parece correcta pero modifica la variable equivocada produce un bucle infinito sin ningún error de sintaxis — el código "se ve bien" pero nunca termina.
for...in con arraysTécnicamente funciona (los índices son claves enumerables), pero no se recomienda: puede iterar propiedades heredadas, no garantiza el orden numérico, y devuelve los índices como strings ("0", "1") en vez de números.
break en bucles anidadosUn break dentro del bucle interno solo rompe el bucle interno, no el externo — para salir de ambos se necesitan técnicas adicionales (variable "bandera" o etiquetas label, poco usadas en código moderno).
continue en forAl usar continue dentro de un for, el incremento (i++) sí se sigue ejecutando antes de volver a evaluar la condición — no te saltas el incremento, solo el resto del cuerpo de esa iteración.
Ternario anidadoEs válido anidar ternarios, pero más de un nivel de anidación reduce mucho la legibilidad; en ese caso se prefiere if/else if/else.

Metáforas y ejemplos visuales

if / else if / elseUn semáforo de decisiones: si está en rojo (condición 1), te detienes; si no, revisas amarillo (condición 2); si tampoco, asumes verde (else) — solo se sigue un camino, nunca varios a la vez.
Operador ternarioUna bifurcación de un solo carril con letrero: "¿lloviendo? → toma paraguas : toma gafas de sol" — una decisión rápida, de una sola línea, que siempre entrega algo.
switchUn casillero de correo con compartimentos etiquetados: la carta (el valor) se coloca directamente en el compartimento (case) que coincide con su etiqueta, sin revisar uno por uno con preguntas de sí/no.
Fall-throughUn casillero de correo sin separadores: si no cierras bien la carta en su compartimento (olvidas el break), se desliza también hacia los compartimentos siguientes.
forUn contador de gimnasio que marca repeticiones: defines desde qué número empieza, hasta dónde llega, y de cuánto en cuánto sube — al llegar al límite, se detiene solo.
whileSeguir sirviendo agua en un vaso mientras no esté lleno: revisas la condición antes de cada vez que sirves; si el vaso ya está lleno desde el inicio, nunca sirves nada.
do...whileProbar un plato de comida antes de decidir si pides más: pruebas primero (el cuerpo se ejecuta sí o sí una vez) y después decides si continúas.
for...ofRepartir un mazo de cartas boca abajo: solo te importa la carta (el valor), no en qué posición del mazo estaba.
for...inRevisar la lista de contenidos de una caja etiquetada (un objeto): te interesan las etiquetas (las claves) para saber qué hay adentro de cada una.
Bucle infinitoUna cinta de correr que nunca se apaga porque el botón de "stop" está desconectado: sigue moviéndose para siempre porque nada detiene la condición.
Bucles anidadosUn reloj: la manecilla de los segundos (bucle interno) da una vuelta completa por cada paso de la manecilla de los minutos (bucle externo).
08

Errores típicos de un junior

  • Olvidar el break en un switch y no entender por qué se ejecutan varios case seguidos (fall-through no intencional).
  • Plantear mal la condición de un bucle (while (i = 5) en vez de while (i === 5), usando asignación en vez de comparación).
  • Confundir for...in con for...of — usar for...in en un array esperando los valores y obtener los índices como strings.
  • Olvidar el i++ (o modificar la variable equivocada) dentro de un while, provocando un bucle infinito.
  • Pensar que continue "detiene todo el bucle" cuando en realidad solo salta a la siguiente iteración.
  • Anidar demasiados ternarios y terminar con una línea imposible de leer, en vez de usar if/else if/else.
  • Usar == en vez de === dentro de condiciones, provocando coerción de tipos no deseada.
  • En bucles anidados, confundir cuál variable de índice pertenece a cuál bucle.
