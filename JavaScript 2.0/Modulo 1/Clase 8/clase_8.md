## Temas que probablemente se cubrieron (usa esto como mínimo, amplía si falta algo lógico)
- Planificación de un proyecto pequeño antes de programar (qué construir, qué funcionalidades)
- Estructura de carpetas de un proyecto frontend simple (`index.html`, `styles.css`, `script.ts`/`script.js`, carpeta `assets`)
- Maquetado HTML del proyecto integrador usando etiquetas semánticas (Clase 4)
- Estilado con Flexbox/Grid para el layout final (Clase 6)
- Lógica de interactividad: eventos, manipulación del DOM (Clase 5)
- Uso de variables, funciones, condicionales y bucles dentro del proyecto real (Clases 2 y 3)
- Tipado con TypeScript aplicado al proyecto (Clase 7) si el proyecto lo incluye
- Conexión correcta entre archivos: `<link>` para CSS, `<script>` para JS/TS (orden de carga, atributo `defer`)
- Buenas prácticas de organización de código (nombres descriptivos, separación de responsabilidades)
- Debugging básico: uso de consola del navegador (`console.log`), DevTools, errores comunes de consola
- Validaciones simples de formulario con JS (si aplica al proyecto)
- Checklist de "proyecto terminado": qué revisar antes de dar por completo un entregable

## Formato de salida requerido (usa EXACTAMENTE esta estructura)

Para el tema completo de la clase, entrega:

1. **Índice de subtemas** cubiertos en la clase de 3h (lista completa y ordenada)
2. **Explicación detallada por subtema** (no solo definición — explica el "por qué" y "cómo funciona", enfatizando cómo se conecta con clases anteriores)
3. **Conceptos clave** (glosario con término + definición corta)
4. **¿Para qué sirve cada concepto?** en la práctica real
5. **Ejemplos abstractos** usando notación `[archivo_1]`, `[funcion_1]`, `[elemento_1]`, indicando cuáles son:

6. **Excepciones y casos especiales** (qué pasa si el nombre de archivo referenciado no coincide, orden incorrecto de carga de script, errores silenciosos comunes en integración)
7. **Metáfora o ejemplo visual** para el proceso de integración completo
8. **Errores típicos** que comete un junior al integrar todo (desorganización de archivos, mezclar lógica y estilos, no probar en el navegador tras cada cambio)
9. **Preguntas de autoevaluación** (mínimo 8) tipo "¿Puedo explicar...?" para que yo verifique si realmente entendí
10. **Checklist final** de dominio del tema (formato casillas ✅), incluyendo un checklist específico de "requisitos mínimos de un proyecto integrador aprobado"
